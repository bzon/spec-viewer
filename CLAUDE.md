# spec-viewer

Local markdown spec viewer for agent-driven development. Go binary serves markdown files in the browser with syntax highlighting, TOC navigation, and configurable themes.

## Design Context

### Users
Developers reading long specification and plan documents generated by AI coding agents. They spend extended time reading these documents and need a comfortable, distraction-free experience. The viewer is a local dev tool, not a public-facing product.

### Brand Personality
Calm, focused, minimal. The interface should disappear — the content is the product. Like opening a well-typeset book: you notice the quality subconsciously but your attention stays on the words.

### Aesthetic Direction
- **Reference:** Notion's document reading experience — clean, spacious, comfortable. Content breathes. Nothing competes for attention.
- **Anti-references:** Cluttered dashboards (too many competing elements), generic markdown renderers (unstyled, no personality), overly designed apps (style over substance, visual noise).
- **Theme:** Dark mode default (github-dark). Light mode available. Five built-in themes with custom theme support.
- **Typography:** Inter for prose, JetBrains Mono for code. Both embedded — no CDN dependency for fonts.

### Design Principles
1. **Content is king** — every UI element exists to serve the document, not to decorate the screen. If it doesn't help reading, remove it.
2. **Quiet confidence** — high quality in typography, spacing, and color, but never showy. The reader should feel the care without noticing it.
3. **Stay out of the way** — sidebars, navigation, and controls are secondary. They're there when needed, invisible when not.
4. **Respect the reader's time** — fast load, instant theme switching, live reload. No loading spinners that last more than a moment. No unnecessary transitions.
5. **Works everywhere it needs to** — keyboard accessible, readable at any text size, functional offline (except syntax highlighting CDN dependency).
